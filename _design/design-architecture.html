Key Design Decisions

  • Flat root — entry points (.bash_profile, .bashrc) stay in root for symlinking
  • core/ is ~200 lines total — just enough to bootstrap the module system
  • modules/ are fully self-contained — add or remove a directory, nothing else changes
  • platform/ is separate from modules — core platform differences (clipboard, open command) don't belong in a "module"
  • bin/ ships pre-built — zero build dependencies on target machines
  • overrides/ is tracked — your preferences travel with the repo
  • Local overrides live at ~/.dotfiles/local/ — untracked, per-machine
