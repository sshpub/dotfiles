Shell Wrapper

bin/dotfiles — the entry point on every machine
#!/usr/bin/env bash # Auto-detect platform/arch then run the right binary # This is the ONLY thing that needs to run on a fresh machine DOTFILES_DIR="$(cd "$(dirname "$0")/.." && pwd)" OS="$(uname -s | tr '[:upper:]' '[:lower:]')" ARCH="$(uname -m)" case "$OS" in darwin) OS="darwin" ;; linux) OS="linux" ;; esac case "$ARCH" in x86_64) ARCH="amd64" ;; aarch64|arm64) ARCH="arm64" ;; esac BINARY="${DOTFILES_DIR}/bin/dotfiles-${OS}-${ARCH}" if [[ ! -x "$BINARY" ]]; then echo "No binary for ${OS}/${ARCH}. Run: cd cli && make build-all" exit 1 fi exec "$BINARY" "$@"

Binary Distribution

Build and ship pre-compiled binaries
Build targets (in cli/Makefile): dotfiles-darwin-arm64 # macOS Apple Silicon dotfiles-darwin-amd64 # macOS Intel dotfiles-linux-arm64 # Linux ARM dotfiles-linux-amd64 # Linux x86_64 (also WSL) Build workflow: 1. Develop in cli/ 2. make build-all # Cross-compile all 4 targets 3. Binaries land in bin/ # Committed to repo 4. git push # Machines get new binary on next pull Why commit binaries? - Zero build deps on target machines (no Go toolchain needed) - git pull = instant update - Binaries are ~5-8MB each (Go static linking) - 4 binaries = ~25MB total, acceptable for a dotfiles repo Future option: GitHub Releases + auto-download on update (but committed binaries are simpler for now)

Module Registries

Configurable registries — public and private
In ~/.dotfiles.json or repo config: { "registries": [ { "name": "default", "url": "https://github.com/dotfiles-modules" }, { "_comment": "Company internal modules", "name": "acme-corp", "url": "git@github.com:acme-corp/dotfiles-modules.git", "private": true } ] } Usage: dotfiles module search kubectl # Search all registries dotfiles module add acme-corp/k8s # Install from private registry dotfiles module add community/fzf # Install from public registry How it works: - Registries are git repos containing modules - CLI clones/pulls registry, copies module into modules/ - Private registries use existing SSH/git auth - Businesses maintain curated module sets for teams
