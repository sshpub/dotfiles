Design: AI & Lightweight Shell Context

Section 5 of 6 — Minimal shell for AI tools + scoped CLAUDE.md for development

The Problem

When Claude Code, Gemini CLI, Codex, OpenCode, or Grok spawn a shell to run commands, they inherit your full interactive shell: oh-my-posh prompt rendering, atuin history sync, completions for 20+ tools, lazy-loaded version managers, module aliases... all for what's usually git diff or npm test. This adds latency, wastes tokens on output noise, and can cause unexpected behavior in non-interactive contexts.

Minimal Mode — Built Into the Loader

core/loader.sh — early exit for non-interactive contexts
# Detect AI CLI tools, CI, or explicit minimal mode # Check known env vars that AI tools set DOTFILES_MINIMAL_TRIGGERS=( CLAUDE_CODE CODEX GEMINI_CLI OPENCODE GROK_CLI CI GITHUB_ACTIONS GITLAB_CI ) dotfiles_is_minimal() { [[ "$DOTFILES_MINIMAL" == "true" ]] && return 0 for var in "${DOTFILES_MINIMAL_TRIGGERS[@]}"; do [[ -n "${!var}" ]] && return 0 done return 1 } # In .bash_profile: if dotfiles_is_minimal; then source "$DOTFILES_DIR/core/platform.sh" source "$DOTFILES_DIR/core/path.sh" source "$DOTFILES_DIR/core/exports.sh" # Load any profile-specified minimal extras dotfiles_load_minimal_extras return fi # ...full interactive loading continues below...

Profile-Controlled Minimal Mode

~/.dotfiles.json — fine-tune what AI tools get
{ "minimal_mode": { "_comment": "Controls shell loading for AI tools and CI", "_comment_triggers": "Env vars that activate minimal mode", "env_triggers": [ "CLAUDE_CODE", "CODEX", "GEMINI_CLI", "OPENCODE", "GROK_CLI", "CI", "GITHUB_ACTIONS", "GITLAB_CI" ], "_comment_always": "Always loaded in minimal mode", "always_load": ["platform", "path", "exports"], "_comment_include": "Optional extras for minimal mode", "include_modules": ["git"], "_comment_never": "Never load in minimal mode", "never_load": [ "prompt", "atuin", "ohmyposh", "starship", "completions", "sync-check", "motd" ] } }

What Each Mode Gets

Minimal Mode (AI / CI)

  • Platform detection
  • Correct PATH (tools are findable)
  • Environment variables (EDITOR, GOPATH, etc.)
  • Optionally: git aliases (if configured)
  • Target: <50ms startup

Clean, fast, no noise.

Full Interactive Mode (you)

  • Everything in minimal +
  • All enabled modules loaded
  • Prompt (oh-my-posh / starship / fallback)
  • Atuin history, completions, zoxide
  • Sync check notification
  • Target: <200ms startup

Full experience, your shell.

CLI Commands for Minimal Mode

Manage and test minimal mode
dotfiles minimal show # Show what loads in minimal mode dotfiles minimal test # Spawn a minimal shell to try it out dotfiles minimal add-trigger CURSOR # Add a new AI tool's env var dotfiles minimal include git # Add a module to minimal mode dotfiles minimal exclude git # Remove a module from minimal mode
